The `tmp_path` fixture will clean up the directory eventually, but explicit cleanup should happen in a finally block or via `addFinalizer`.
  Fix: Move the overflow file read and cleanup into a try/finally, or rely on `tmp_path` cleanup (which already happens) and remove the manual unlink.

The decorators are present on all tests, so this works, but adding `asyncio_mode = "auto"` in a config file would be more maintainable and prevent a missed-decorator regression.
  Fix: Add a `pyproject.toml` or `conftest.py` with `asyncio_mode = "auto"` to the session-spawner directory.

The spec says "truncated by byte boundary" which implies multi-byte character handling matters. A test with multi-byte UTF-8 characters at the truncation boundary would verify the `errors="ignore"` decode behavior.
  Fix: Add a test variant (or assertion within this test) that includes multi-byte characters near the 50,000-byte boundary to verify clean truncation.
