# WI-178: Harden pretest fail-fast and add prebuild:migration cleanup step

## Context

This work item addresses Q-68 and Q-71 from cycle 020 review.

**Q-68**: `npm run build:migration` leaves stale `.d.ts` and `.js.map` files on disk from prior `tsc` runs. The `.gitignore` entries added in WI-174 prevent committing them, but they accumulate in contributors' working trees. A `prebuild:migration` npm lifecycle script (auto-run before `build:migration`) should delete them before each compilation.

**Q-71**: The `pretest` script compares `migrate-to-v3.ts` and `migrate-to-v3.js` mtimes and emits a warning to stderr if `.js` is older, but exits 0. This means tests proceed against a stale compiled file with no hard stop. Additionally, if `migrate-to-v3.js` does not exist (fresh clone before running `build:migration`), `statSync` throws and `catch(e) {}` swallows the error silently — no warning at all.

## File to Modify

`mcp/artifact-server/package.json`

### Current scripts section (lines 7-15):

```json
"scripts": {
  "build": "tsc",
  "start": "node dist/index.js",
  "dev": "node --loader ts-node/esm src/index.ts",
  "pretest": "node --input-type=module -e \"import { statSync } from 'fs'; try { const ts=statSync('../../scripts/migrate-to-v3.ts').mtimeMs; const js=statSync('../../scripts/migrate-to-v3.js').mtimeMs; if(js<ts){process.stderr.write('WARNING: migrate-to-v3.js may be stale — run: cd mcp/artifact-server && npm run build:migration\\n');} } catch(e) {}\"",
  "test": "vitest run",
  "build:migration": "tsc ../../scripts/migrate-to-v3.ts --target ES2022 --module NodeNext --moduleResolution NodeNext --esModuleInterop --skipLibCheck --outDir ../../scripts --declaration false",
  "test:coverage": "vitest run --coverage"
}
```

### Changes required:

1. **Add `prebuild:migration`** before `build:migration`. npm automatically runs `prebuild:migration` before `build:migration`. The script should delete the three stale artifact files:

```json
"prebuild:migration": "node -e \"['../../scripts/migrate-to-v3.d.ts','../../scripts/migrate-to-v3.d.ts.map','../../scripts/migrate-to-v3.js.map'].forEach(f=>{try{require('fs').rmSync(f)}catch(e){}})\"",
```

Note: Use `try/catch` around each `rmSync` call so the script is idempotent (no error if files don't exist). This runs as CJS (no `--input-type=module` flag needed since `require` is used).

2. **Modify `pretest`** to:
   - Exit non-zero when staleness is detected (add `process.exit(1)` after the `process.stderr.write` call)
   - Emit a warning (not swallow silently) when `migrate-to-v3.js` is absent — change the `catch(e) {}` to check whether the error is because `.js` is missing vs. some other error

The updated `pretest` one-liner logic:
- Try to stat both files
- If `.js` is absent (ENOENT for the `.js` path): emit a "migrate-to-v3.js not found — run build:migration" warning and exit 1
- If `.js` is stale (js mtime < ts mtime): emit the existing staleness warning and exit 1
- If both exist and `.js` is current: exit 0
- If `.ts` is absent or any other unexpected error: emit the error message and continue (exit 0 — don't block tests on infra issues)

Suggested updated `pretest` one-liner:

```
node --input-type=module -e "import { statSync } from 'fs'; try { const ts=statSync('../../scripts/migrate-to-v3.ts').mtimeMs; let js; try { js=statSync('../../scripts/migrate-to-v3.js').mtimeMs; } catch(e) { if(e.code==='ENOENT'){process.stderr.write('WARNING: migrate-to-v3.js not found — run: cd mcp/artifact-server && npm run build:migration\\n');process.exit(1);} throw e; } if(js<ts){process.stderr.write('WARNING: migrate-to-v3.js may be stale — run: cd mcp/artifact-server && npm run build:migration\\n');process.exit(1);} } catch(e) {}"
```

The outer `catch(e) {}` still silently swallows errors that prevent statting `migrate-to-v3.ts` — this is intentional, as we don't want infra issues (missing .ts file, permission errors) to break test runs.
