# WI-179: Extend toYaml array-item quoting to full parity with scalar guard

## Context

This work item addresses Q-69 from cycle 020 review.

The `toYaml` function in `scripts/migrate-to-v3.ts` has two quoting guards:

1. **Scalar guard** (line ~76–97): applied to map values. Checks 14 conditions.
2. **Array-item guard** (line ~110): applied to string array items. Checks only 5 conditions.

The 9 conditions in the scalar guard not present in the array-item guard:
- Starts with `>` (YAML block scalar indicator)
- Starts with `'` (YAML single-quoted scalar)
- Starts with `{` (YAML inline mapping)
- Starts with `[` (YAML inline sequence)
- Starts with `*` (YAML alias)
- Starts with `&` (YAML anchor)
- Starts with `!` (YAML tag)
- Starts with `|` (YAML literal block scalar)
- Contains `\t` (tab character)
- Starts with a digit
- Equals YAML keyword: `true`, `false`, `null`, `yes`, `no`, `on`, `off`

(That is 11 individual conditions, but organized into 3 groups above plus the digit/keyword checks.)

### 3. `mcp/artifact-server/src/__tests__/migrate.test.ts`

Add test cases to the existing `toYaml` describe block. The test file already imports `toYaml` from the compiled `migrate-to-v3.js`. Add assertions for 4 representative new conditions:

```typescript
it("quotes array items that are YAML boolean keywords", () => {
  const result = toYaml({ items: ["true"] });
  expect(result).toContain('- "true"');
});

it("quotes array items starting with a YAML indicator character", () => {
  const result = toYaml({ items: ["{key: val}"] });
  expect(result).toContain('- "{key: val}"');
});

it("quotes array items starting with a digit", () => {
  const result = toYaml({ items: ["123abc"] });
  expect(result).toContain('- "123abc"');
});

it("quotes array items containing a tab character", () => {
  const result = toYaml({ items: ["col1\tcol2"] });
  expect(result).toContain('- "col1\\tcol2"');
});
```

Note on the tab test: `toYaml` escapes `\` to `\\` and `"` to `\"`. A literal tab `\t` in the string is not escaped by the current escaping logic — it would be emitted as a literal tab inside double-quotes. YAML allows literal tab characters inside double-quoted scalars, so this is valid. The assertion should check that the item is quoted (wrapped in `"`), not that the tab is escaped.

Revised tab test:
```typescript
it("quotes array items containing a tab character", () => {
  const result = toYaml({ items: ["col1\tcol2"] });
  // Array item should be wrapped in quotes; the tab is emitted as-is inside the quoted scalar
  expect(result).toMatch(/- "col1\tcol2"/);
});
```
