# WI-174: Add build:migration npm script to package.json

## Context

`migrate-to-v3.ts` (project root `scripts/`) is compiled to `migrate-to-v3.js` by hand. Both files exist and are kept in sync manually. WI-172 added a `pretest` staleness warning and documented the recovery command as `npm run build:migration`. That script was never added to `package.json`, breaking the documented recovery path (Q-63, cycle 019 significant finding).

## File to Modify

`mcp/artifact-server/package.json` — add one entry to the `scripts` object.

## Current State

```json
"scripts": {
  "build": "tsc",
  "start": "node dist/index.js",
  "dev": "node --loader ts-node/esm src/index.ts",
  "pretest": "node --input-type=module -e \"...staleness check...\"",
  "test": "vitest run",
  "test:coverage": "vitest run --coverage"
}
```

## What to Add

Add `"build:migration"` to the scripts object. The command must compile `scripts/migrate-to-v3.ts` (at `../../scripts/migrate-to-v3.ts` relative to `mcp/artifact-server/`) to `scripts/migrate-to-v3.js` (at `../../scripts/migrate-to-v3.js`).

The TypeScript compiler settings must match the ESM build: `--target ES2022 --module NodeNext --moduleResolution NodeNext --esModuleInterop --skipLibCheck`.

A working invocation from `mcp/artifact-server/`:
```
npx tsc --target ES2022 --module NodeNext --moduleResolution NodeNext --esModuleInterop --skipLibCheck --outDir ../../scripts ../../scripts/migrate-to-v3.ts
```

With `--outDir ../../scripts` and a single input file, tsc places the output at `../../scripts/migrate-to-v3.js`. Verify the command runs without error and produces the expected file before committing.

Note: `--declaration false` may be needed to suppress `.d.ts` generation in the scripts directory. Check the output and add it if `.d.ts` files are produced alongside the `.js`.

## Verification

After implementing:
1. Run `npm run build:migration` from `mcp/artifact-server/` — must exit 0
2. Verify `scripts/migrate-to-v3.js` exists and its mtime is newer than `scripts/migrate-to-v3.ts`
3. Run `npm test` — pretest staleness warning must not fire (JS is now fresh)
