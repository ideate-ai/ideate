# WI-177: Add checkSchemaVersion version-0 path test

## Context

`checkSchemaVersion` in `mcp/artifact-server/src/schema.ts` treats `user_version = 0` as "fresh DB — compatible" and returns `true` without deleting the file. This is the correct behavior for a freshly created SQLite file (SQLite sets `user_version = 0` by default). The branch at line 601 is correct but untested (Q-67).

The gap: no test asserts this path. If a future refactor changed the version-0 branch (e.g., treating 0 as "unknown/corrupt" instead of "fresh"), no test would catch the regression.

## File to Modify

`mcp/artifact-server/src/__tests__/schema.test.ts` — add one test to the existing `checkSchemaVersion` describe block.

## What to Add

```typescript
it("returns true for a fresh database with user_version = 0", () => {
  const db = new Database(":memory:");
  // SQLite sets user_version = 0 by default on a new database
  const result = checkSchemaVersion(db, "/nonexistent/path.db");
  expect(result).toBe(true);
  db.close();
});
```

Key points:
- Use `":memory:"` as the database path to avoid touching the filesystem
- Pass a non-existent path string as the `dbPath` argument — the version-0 branch does not call `fs.rmSync`, so this string is never used
- The test asserts `true` is returned
- `db.close()` in the happy path (version-0 branch does NOT close the db, unlike the version-mismatch branch) — close it manually in the test to avoid open handle warnings

Check the existing `schema.test.ts` for imports and test helpers already in use before writing the test.
