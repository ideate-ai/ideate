# WI-175: Fix toYaml array-item whitespace guard and clean up stale test call sites

## Context

Two independent issues, both in files owned by this work item:

**Issue A ‚Äî array-item whitespace guard (Q-66)**: `toYaml` in `migrate-to-v3.ts` has a scalar-value quoting guard at line 76 that checks `/^\s/.test(value)` (leading whitespace). The parallel array-item branch at line 110 checks `\n`, `"`, `:`, `#` but NOT `/^\s/`. A string array item with leading whitespace is emitted unquoted, producing invalid YAML. The same gap exists in `migrate-to-v3.js`.

**Issue B ‚Äî stale 3-arg call sites (Q-65)**: WI-172 removed the `_ideateDir` third parameter from `migratePlanArtifacts`, `migrateSteeringArtifacts`, and `migrateInterviews`. The function signatures now accept 2 parameters (`ctx`, `sourceDir`). Approximately 10 test call sites in `migrate.test.ts` still pass a third argument (`ideateDir`) that is silently discarded by JavaScript. TypeScript does not catch this because the test imports from the `.js` file.

## Files to Modify

### scripts/migrate-to-v3.ts ‚Äî array-item branch fix

At line 110, the condition currently reads:
```typescript
if (item.includes("\n") || item.includes('"') || item.includes(":") || item.includes("#")) {
```

Change to:
```typescript
if (item.includes("\n") || item.includes('"') || item.includes(":") || item.includes("#") || /^\s/.test(item)) {
```

No other changes to this file.

### scripts/migrate-to-v3.js ‚Äî mirror the same fix

Find the corresponding line in the compiled JavaScript (the array-item quoting condition). Apply the identical fix: add `|| /^\s/.test(item)` to the condition.

Use the `.ts` change as the reference. Verify both files are in sync after editing.

### mcp/artifact-server/src/__tests__/migrate.test.ts

**Fix A ‚Äî add whitespace test**: In the `toYaml` describe block, add a test case:
```typescript
it("quotes array items with leading whitespace", () => {
  const result = toYaml({ items: [" indented"] });
  expect(result).toContain('"');
});
```
(The exact assertion may vary ‚Äî what matters is that the output contains a quoted form, not bare `- " indented"` without quotes.)

**Fix B ‚Äî remove spurious third arguments**: Find all call sites for `migratePlanArtifacts`, `migrateSteeringArtifacts`, and `migrateInterviews`. Each currently passes 3 arguments: `(ctx, tmpSrc, ideateDir)`. Remove the third argument from every call site. The corrected form is `(ctx, tmpSrc)`.

From the grep results, the affected call sites are approximately at lines 852, 881, 923, 959, 990, 1034, 1063, 1089, 1121, 1166, 1183, 1193, 1203. Verify by searching for the three function names ‚Äî do not miss any.
