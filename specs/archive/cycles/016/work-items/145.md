## Files to modify

### tools.ts
Strip all 7 tool definitions and the dispatch function. Export an empty tools array. The MCP server runs but exposes no tools — tools are added in Phase 2.

### indexer.ts — Complete rewrite
Current indexer.ts contains implementations for the 7 read tools. Replace entirely with:

```typescript
export interface RebuildStats {
  files_scanned: number;
  files_updated: number;
  files_deleted: number;
  edges_created: number;
  cycles_detected: string[][];
}

export function rebuildIndex(db: Database, ideateDir: string): RebuildStats {
  // 1. Scan all YAML files in .ideate/ subdirectories
  // 2. For each file:
  //    a. Compute SHA-256 content hash
  //    b. Compare with stored content_hash — skip if unchanged
  //    c. Parse YAML
  //    d. Map to typed table row using schema.ts mappings
  //    e. UPSERT into the appropriate table
  //    f. Extract inline edges, UPSERT into edges table
  //    g. Extract file refs, UPSERT into node_file_refs
  // 3. Delete rows for files that no longer exist on disk
  // 4. Run DAG cycle detection (Kahn's algorithm on depends_on edges)
  // 5. Return stats
}
```

The rebuild function runs in a single SQLite transaction for atomicity.

### Kahn's algorithm for DAG validation

```typescript
function detectCycles(db: Database): string[][] {
  // 1. SELECT all depends_on edges from edges table
  // 2. Build adjacency list and in-degree map
  // 3. Initialize queue with zero-in-degree nodes
  // 4. Process queue (standard Kahn's)
  // 5. Nodes with remaining non-zero in-degree are in cycles
  // 6. Return array of cycle arrays (empty if no cycles)
}
```

Log warnings for detected cycles but do not fail the rebuild.

### watcher.ts
Update to watch `.ideate/` directory for YAML file changes. On change, trigger incremental rebuild (only the changed file). On delete, trigger removal of the file's rows from all tables.

### index.ts
Update startup sequence:
1. Resolve .ideate/ directory using config.ts
2. Create/open SQLite database at `.ideate/index.db`
3. Set WAL mode: `PRAGMA journal_mode=WAL`
4. Set busy timeout: `PRAGMA busy_timeout=5000`
5. Call createSchema(db) from schema.ts
6. Call rebuildIndex(db, ideateDir)
7. Start file watcher
8. Start MCP server (no tools registered)

### package.json
Remove `@xenova/transformers` from dependencies. It was used only by embeddings.ts. All other deps (better-sqlite3, chokidar, lru-cache, @modelcontextprotocol/sdk) are retained. Evaluate whether lru-cache is still needed — if the rebuild pipeline doesn't benefit from caching, remove it.

## Performance considerations

- Content-hash comparison avoids reparsing unchanged files
- Single-transaction rebuild prevents partial state
- WAL mode allows concurrent reads during rebuild
- Incremental updates (watcher-triggered) reprocess only the changed file
- Kahn's algorithm is O(V+E) — negligible for hundreds of work items
