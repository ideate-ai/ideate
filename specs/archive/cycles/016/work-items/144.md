# WI-144: YAML artifact schemas and SQLite runtime index schema

## Artifact types and YAML file layout

Each artifact is a single YAML file. No markdown bodies — all content is structured YAML fields.

### WorkItem (work-items/WI-NNN.yaml)

## SQLite schema

Typed node tables + universal edges table (Option 1 from artifact-graph-schema.md).

Common columns on every typed table:
- `id TEXT PRIMARY KEY`
- `type TEXT NOT NULL` (redundant with table name, useful for edge joins)
- `cycle_created INTEGER`
- `cycle_modified INTEGER`
- `content_hash TEXT NOT NULL`
- `token_count INTEGER`
- `file_path TEXT NOT NULL`
- `status TEXT`

Universal edges table:
```sql
CREATE TABLE edges (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  source_id TEXT NOT NULL,
  source_type TEXT NOT NULL,
  target_id TEXT NOT NULL,
  target_type TEXT NOT NULL,
  edge_type TEXT NOT NULL,
  props TEXT,
  UNIQUE(source_id, target_id, edge_type)
);
CREATE INDEX idx_edges_source ON edges(source_id, edge_type);
CREATE INDEX idx_edges_target ON edges(target_id, edge_type);
CREATE INDEX idx_edges_composite ON edges(source_id, target_id, edge_type);
```

Bridge table for file references:
```sql
CREATE TABLE node_file_refs (
  node_id TEXT NOT NULL,
  node_type TEXT NOT NULL,
  file_path TEXT NOT NULL,
  PRIMARY KEY (node_id, file_path)
);
CREATE INDEX idx_file_refs_path ON node_file_refs(file_path);
```

## Edge type enumeration

| Edge type | Source | Target | Extracted from YAML field |
|---|---|---|---|
| depends_on | work_item | work_item | WorkItem.depends |
| blocks | work_item | work_item | WorkItem.blocks |
| belongs_to_module | work_item | module_spec | WorkItem.module |
| belongs_to_domain | policy/decision/question | domain (string) | *.domain |
| derived_from | domain_policy | guiding_principle | DomainPolicy.derived_from |
| relates_to | finding | work_item | Finding.work_item |
| addresses | work_item | finding/question | explicit reference |
| supersedes | domain_decision | domain_decision | DomainDecision.supersedes |
| amends | domain_policy | domain_policy | amendment chain |

## Implementation notes

- Use better-sqlite3 synchronous API for all operations
- WAL mode for concurrent read access
- Content hash is SHA-256 of the full YAML file content
- token_count estimated by character count / 4 (rough approximation, no tokenizer dependency)
- All tables created in a single transaction via createSchema(db)
- Export TypeScript interfaces matching each table's columns
- Export field mapping functions: YAML field name → SQLite column name per artifact type
