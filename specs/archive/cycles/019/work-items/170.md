# WI-170: Watcher debounce

## Problem

Each `change`/`add`/`unlink` event from chokidar triggers a full `rebuildIndex()` call immediately. When a skill or migration script writes 10+ YAML files in rapid succession, each write fires a separate event, producing N rebuilds instead of 1. Cycle 018 code review flagged this as Significant (S1).

## Implementation

Add a 500ms trailing debounce to the event handler in `ArtifactWatcher`.

**Approach**: Use a `setTimeout`/`clearTimeout` pattern (no external dependencies).

```typescript
// In watcher.ts ArtifactWatcher class or start() method:
let debounceTimer: ReturnType<typeof setTimeout> | null = null;

const debouncedCallback = () => {
  if (debounceTimer) clearTimeout(debounceTimer);
  debounceTimer = setTimeout(() => {
    debounceTimer = null;
    onRebuild(); // the actual rebuildIndex call
  }, 500);
};

watcher.on('change', debouncedCallback)
       .on('add', debouncedCallback)
       .on('unlink', debouncedCallback);
```

The 500ms window is sufficient to coalesce burst writes from any skill execution. The rebuild is idempotent so there is no correctness risk.
