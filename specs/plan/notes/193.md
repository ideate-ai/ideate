# WI-193: Add depth > 1 graph traversal test

## Test Design

Add a new test to `mcp/artifact-server/src/__tests__/tools.test.ts` inside the `handleArtifactQuery` describe block.

### Setup
1. Create 3 nodes in the test DB: `WI-TEST-A`, `WI-TEST-B`, `WI-TEST-C` (all type `work_item`)
2. Create 2 depends_on edges: A→B, B→C
3. Create corresponding work_items extension rows (title required)

### Test: "traverses dependency chain at depth > 1"
```typescript
const result = await handleArtifactQuery(ctx, {
  related_to: "WI-TEST-A",
  edge_types: ["depends_on"],
  direction: "outgoing",
  depth: 3,
});
// Assert: result contains WI-TEST-B (depth 1) and WI-TEST-C (depth 2)
// Assert: no duplicate rows
// Assert: result is a markdown table with Edge, Dir, Depth columns
```

### Why this catches S1 and S2
- S1 (cycle protection): If a cycle existed (A→B→A), UNION ALL would produce infinite rows up to depth limit. UNION deduplicates.
- S2 (ambiguous id): The ORDER BY uses the column from the CTE output. If it's ambiguous, SQLite throws an error or sorts wrong.
