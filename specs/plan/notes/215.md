# WI-215: Cycle Reporting Script

## Design

`scripts/report-cycle.sh` — focused on cycle-over-cycle quality trends.

Uses the same bash-wrapping-python pattern as `scripts/report.sh`. Reuse the same auto-discovery logic and helper functions (fmt_tokens, fmt_ms).

### Sections:

1. **Cycle-over-Cycle Quality Trends** — Same as report.sh section_quality_trends but as the primary output. Per-cycle finding counts by severity with trend indicator.

2. **Convergence Speed** — Read convergence_cycles from metrics entries with event_type "convergence_summary" or "cycle_complete". If not available, count distinct quality_summary events per cycle as a proxy.

3. **First-Pass Acceptance Rate** — For each cycle, count work items where first_pass_accepted is true vs total. Display as percentage. If first_pass_accepted field is not present in any entry, display "No first-pass data recorded" message.

### Auto-discovery
Same pattern as report.sh: walk up from CWD looking for .ideate.json, read artifactDir, append /metrics.jsonl.
