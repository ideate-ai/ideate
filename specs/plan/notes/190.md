# WI-190: Test Suite Rewrite

## Scope

Rewrite `schema.test.ts` and `indexer.test.ts` for v7 schema. Create `tools.test.ts` for all 11 tools. Leave `config.test.ts`, `watcher.test.ts`, and `migrate.test.ts` unchanged.

## schema.test.ts

### Tests to rewrite
- `creates all tables`: verify `nodes` table exists with 8 columns + 12 extension tables exist + `edges` and `node_file_refs` exist
- `sets schema version`: verify `user_version = 7`
- `creates indexes`: verify idx_nodes_type, idx_nodes_file_path, idx_edges_source, idx_edges_target, idx_file_refs_path, plus type-specific indexes

### New tests to add
- `FK constraint: extension table rejects orphan insert`: INSERT into work_items with id not in nodes â†’ expect FOREIGN KEY constraint error
- `ON DELETE CASCADE: deleting from nodes cascades to extension table`: INSERT node + work_item, DELETE from nodes, verify work_items row is gone
- `ON DELETE CASCADE: deleting from nodes cascades to edges`: INSERT two nodes + edge between them, DELETE source node, verify edge is gone
- `ON DELETE CASCADE: deleting from nodes cascades to node_file_refs`: INSERT node + file_ref, DELETE from nodes, verify file_ref is gone
- `edges table has no source_type or target_type columns`: verify column list
- `checkSchemaVersion tests`: preserved from v6 (version-0, version-mismatch, version-current)

## indexer.test.ts

### Tests to rewrite
- `rebuildIndex populates tables`: verify both nodes AND extension table have rows
- `rebuildIndex respects content hash (skip unchanged)`: verify hash check queries nodes table only
- `rebuildIndex deletes stale rows via CASCADE`: insert a node+extension+edge, then run rebuild without that file, verify all three are deleted

### Tests to add
- `rebuildIndex uses deferred FK`: create a YAML file with edges referencing other nodes, verify edges are valid after rebuild completes (FK not violated)
- `buildNodeRow returns 8-column object`: unit test the row builder
- `buildExtensionRow returns type-specific columns`: unit test per type

- Call `ideate_artifact_query` with `type: "journal_entry"`
- Verify the new entry appears in the query results (confirms sync SQLite update)
