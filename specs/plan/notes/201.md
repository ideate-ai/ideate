## Audit criteria

For each file, check:

1. **GP-8 / P-6 / P-26 / P-32 (MCP mandatory)**:
   - Does the file contain "MCP availability check" patterns for ideate artifact server tools? These should be REMOVED — the artifact server is always present (P-32).
   - Does the file instruct agents to read YAML files directly? This should be changed to MCP tool calls (P-6).
   - Does the file instruct agents to write YAML files directly? Writes should go through MCP write tools where they exist (P-26).
   - MCP availability checks for EXTERNAL tools (e.g., outpost session spawner) should be PRESERVED.

2. **GP-13 / P-8 (validation strategy)**:
   - Do acceptance criteria templates assume machine-verifiable only? Update to acknowledge human-in-the-loop validation for subjective criteria.
   - Does the code-reviewer agent handle subjective criteria appropriately?

3. **P-19 (canonical paths)**:
   - Do agent definitions reference `.ideate/` paths correctly?
   - Any stale references to `archive/incremental/`, `reviews/incremental/`, or other pre-v3 paths?

4. **P-14 (outpost references)**:
   - Any mentions of "outpost" that should be generalized to "external MCP servers"?
   - Any `spawn_session` references that should be noted as external-only?

5. **P-10 (non-overlapping responsibilities)**:
   - Do agent boundaries still make sense with MCP tools handling data access?

## Output format

Write findings to `specs/archive/cycles/026/audit-findings.md`:

```markdown
# LLM Artifact Audit — Cycle 026

## Must-Fix Findings
### AF-{N}: {title}
- **File**: {path}:{line range}
- **Current**: {what it says now}
- **Required**: {what it should say}
- **Principle**: {which GP/P it violates}

## Should-Fix Findings
### AF-{N}: ...

## Defer Findings
### AF-{N}: ...
```

Each must-fix finding becomes a work item (WI-202+). Should-fix findings may be bundled into work items or deferred. Defer findings are documented but not actioned this cycle.

## Agent model

Use opus for this audit — it requires cross-cutting judgment across many files and must accurately identify which "MCP availability check" patterns are for the ideate server (remove) vs external tools (keep).
