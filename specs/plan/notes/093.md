# Implementation Notes: 093 — Quality summary event from review skill

## Placement

Add a new section to `skills/review/SKILL.md` between:
- The domain curator phase (last phase before the final summary)
- The final summary presentation to the user

Title it `## Quality Metrics Event` (or match the existing phase numbering convention).

## Event schema

```json
{
  "timestamp": "<ISO8601>",
  "event_type": "quality_summary",
  "skill": "review",
  "cycle": <N>,
  "findings": {
    "total": <N>,
    "by_severity": {"critical": <N>, "significant": <N>, "minor": <N>, "suggestion": <N>},
    "by_reviewer": {
      "code-reviewer": {"critical": <N>, "significant": <N>, "minor": <N>},
      "spec-reviewer": {"critical": <N>, "significant": <N>, "minor": <N>},
      "gap-analyst": {"critical": <N>, "significant": <N>, "minor": <N>}
    },
    "by_category": {
      "requirements_missed": <N>,
      "bugs_introduced": <N>,
      "principles_violated": <N>,
      "implementation_gaps": <N>,
      "other": <N>
    }
  },
  "work_items_reviewed": <N>,
  "andon_events": <N>
}
```

## Derivation logic

The review skill already produces `{output-dir}/summary.md` with severity-grouped finding lists using `[{reviewer}]` prefixes. Parse these:

1. **by_severity**: Count findings under each severity heading
2. **by_reviewer**: Parse `[code-reviewer]`, `[spec-reviewer]`, `[gap-analyst]` prefixes
3. **by_category**:
   - `requirements_missed`: gap-analyst findings tagged as missing requirements (MR sections in gap-analysis.md)
   - `bugs_introduced`: code-reviewer critical + significant findings
   - `principles_violated`: spec-reviewer findings from "Principle Violations" section
   - `implementation_gaps`: gap-analyst minor findings + incomplete integration findings
   - `other`: everything else
4. **work_items_reviewed**: Count distinct work items in incremental reviews
5. **andon_events**: Count from journal.md entries for this cycle mentioning Andon, or 0
6. **suggestion**: Count items under "Suggestions" heading in summary.md

## Cycle number

Derive from `domains/index.md` `current_cycle` value (just updated by curator), or from the `archive/cycles/{NNN}/` directory number.

## Best-effort clause

If derivation or write fails at any step, skip the event and continue. Do not block the review. Log: "quality_summary event skipped: {reason}" in the final summary output.
