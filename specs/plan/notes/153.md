# WI-153: Update edge type registry for reversed-direction edges

## Context

D-59 established that `addresses` and `amends` should be reversed to `addressed_by` and `amended_by` — scalar fields on the target artifact (finding, domain_policy) rather than arrays on the source (work_item, domain_policy). See P-29.

## Implementation

### schema.ts — EDGE_TYPES and EDGE_TYPE_REGISTRY

Replace `"addresses"` with `"addressed_by"` and `"amends"` with `"amended_by"` in the `EDGE_TYPES` array.

Update the registry entries:

```ts
addressed_by: {
  description: "Finding or question has been addressed by a work item",
  source_types: ["finding", "domain_question"],
  target_types: ["work_item"],
  yaml_field: "addressed_by",
},
amended_by: {
  description: "Domain policy has been superseded by a newer version",
  source_types: ["domain_policy"],
  target_types: ["domain_policy"],
  yaml_field: "amended_by",
},
```

Note the direction flip: `source_types` is now the artifact that carries the field (finding/domain_policy), and `target_types` is what it points to (work_item/domain_policy).

### indexer.ts — extractEdges

No code changes needed — `extractEdges` already iterates the registry and extracts based on `yaml_field` and `source_types`. The reversed direction is handled purely by the registry metadata change. The `extractEdges` function will now:
- For a finding YAML with `addressed_by: WI-150`, create edge `(F-016-001, finding) → (WI-150, work_item)` with type `addressed_by`
- For a policy YAML with `amended_by: P-013`, create edge `(P-012, domain_policy) → (P-013, domain_policy)` with type `amended_by`
