## index.ts Changes

### Startup Pragmas
Add immediately after `new Database(dbPath)`:
```typescript
db.pragma('foreign_keys = ON');  // CRITICAL — enables FK enforcement
```

### Tool Registration
Replace current TOOLS import and handleTool call with:
```typescript
import { TOOLS, handleTool } from "./tools/index.js";

// In CallToolRequestSchema handler:
const ctx: ToolContext = { db, drizzleDb, ideateDir };
try {
  const result = await handleTool(ctx, request.params.name, request.params.arguments ?? {});
  return { content: [{ type: "text", text: result }] };
} catch (err) {
  return { content: [{ type: "text", text: String(err) }], isError: true };
}
```

### Remove Old tools.ts
Delete `src/tools.ts` (replaced by `src/tools/` directory).

## Tool Definition Format

Each tool group file exports its tools as an array of `Tool` objects:
```typescript
export const contextTools: Tool[] = [
  {
    name: "ideate_get_work_item_context",
    description: "...",
    inputSchema: { type: "object", properties: { artifact_dir: {...}, work_item_id: {...} }, required: [...] }
  },
  // ...
];
```

The inputSchema for all 11 tools must be defined in this WI even though the handlers are stubs. The actual handler implementations are in WI-185 through WI-189. This WI creates the skeleton; those WIs fill in the logic.

## Error Handling Pattern

Tools throw on error. The MCP handler catches and returns `isError: true`. Standard across all 11 tools. No try/catch inside individual tool handlers unless they need to translate a specific error.
