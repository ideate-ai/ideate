# WI-163: Migration ‚Äî plan + steering artifacts

## Context

`runMigration` in `scripts/migrate-to-v3.ts` (lines ~1157-1165) calls 9 migration functions but does not migrate plan artifacts or steering artifacts. The gap-analyst identified this as MR1: after running the migration, the `.ideate/` directory has no plan or steering artifacts ‚Äî any MCP query for architecture, principles, or constraints returns nothing.

WI-146 specification (the original migration work item) listed these as required migration steps.

## What to migrate

### Plan artifacts

From `specs/plan/` (relative to the source ideate project root):
- `plan/architecture.md` ‚Üí `.ideate/plan/architecture.yaml` with `type: architecture`
- `plan/overview.md` ‚Üí `.ideate/plan/overview.yaml` with `type: overview`
- `plan/execution-strategy.md` ‚Üí `.ideate/plan/execution-strategy.yaml` with `type: execution_strategy`
- `plan/modules/*.md` ‚Üí `.ideate/plan/modules/{name}.yaml` each with `type: module_spec`

### Steering artifacts

From `specs/steering/`:
- `steering/guiding-principles.md` ‚Üí `.ideate/steering/guiding-principles.yaml` with `type: guiding_principles`
- `steering/constraints.md` ‚Üí `.ideate/steering/constraints.yaml` with `type: constraints`
- `steering/research/*.md` ‚Üí `.ideate/steering/research/{name}.yaml` each with `type: research`

## YAML output format

Each migrated file is a YAML object with these fields:

```yaml
id: {type}-{slug}          # e.g., "architecture", "overview", "constraints"
type: {type}               # e.g., "architecture", "guiding_principles"
title: {derived from filename or first heading}
content: |
  {raw markdown content of the source file}
content_hash: {sha256}
token_count: {approx}
```

Use the existing `buildArtifact(canonical)` helper for `content_hash` and `token_count`.

For `plan/modules/{name}.md`, derive `id` as `module_spec-{name}` (slug the filename without extension).

## Handling missing files

Not all plan artifacts will exist in every project. If a file does not exist, skip it silently (log a debug message if verbose). Do not fail the migration if `plan/modules/` is empty or does not exist.

## Call sites in runMigration

Add after the existing migration calls:
```typescript
await migratePlanArtifacts(ctx, sourceDir, ideateDir);
await migrateSteeringArtifacts(ctx, sourceDir, ideateDir);
```
