# WI-154: Integrate Drizzle ORM — replace raw SQL in indexer.ts

## Context

D-63 decided on Drizzle ORM for type-safe query building. Code-reviewer S3 flagged `upsertRow` interpolating column names into SQL as a latent injection risk. Drizzle eliminates this by construction.

## Implementation

### package.json

Add `drizzle-orm` to dependencies. No additional dialect package needed — Drizzle has built-in `better-sqlite3` support.

### src/db.ts (new file)

Define Drizzle table objects matching every table in `schema.ts`. Use `sqliteTable()` from `drizzle-orm/sqlite-core`:

```ts
import { sqliteTable, text, integer } from "drizzle-orm/sqlite-core";

export const workItems = sqliteTable("work_items", {
  id: text("id").primaryKey(),
  type: text("type"),
  // ... all columns from schema.ts
});

export const findings = sqliteTable("findings", {
  id: text("id").primaryKey(),
  // ... including addressed_by: text("addressed_by")
});

// ... all 12 typed tables + edges + node_file_refs
```

The DDL in `schema.ts` remains the authority for CREATE TABLE statements. The Drizzle table definitions are a parallel type-safe layer for queries only — they must match the DDL exactly.

### src/indexer.ts

Replace raw SQL functions:

1. `upsertRow` — replace `db.prepare(\`INSERT INTO ${table}...\`)` with Drizzle's `db.insert(tableRef).values(row).onConflictDoUpdate({ target: tableRef.id, set: row })`
2. `upsertEdge` — replace with Drizzle insert on edges table with `.onConflictDoNothing()`
3. `upsertFileRef` — replace with Drizzle insert on node_file_refs with `.onConflictDoNothing()`
4. `deleteStaleRows` — replace per-row DELETE with Drizzle `db.delete(tableRef).where(notInArray(tableRef.file_path, [...existingPaths]))`
5. SELECT queries in `deleteStaleRows` and hash lookup — replace with Drizzle select

The `upsertRow` function signature changes from `(db, table: string, row: Row)` to `(drizzleDb, tableRef, row)` where `tableRef` is a Drizzle table object. No more string table names. A mapping from `TYPE_TO_TABLE` string values to Drizzle table objects replaces the current string-based dispatch.

### src/index.ts

Wrap the `better-sqlite3` database with Drizzle:

```ts
import { drizzle } from "drizzle-orm/better-sqlite3";
const drizzleDb = drizzle(db);
```

Pass `drizzleDb` to `rebuildIndex` instead of (or alongside) `db`.

### Testing

All existing tests should pass without modification — Drizzle is an internal implementation change. Tests use `better-sqlite3` directly for setup/assertions; the indexer's internal query mechanism changes but external behavior does not.
