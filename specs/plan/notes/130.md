## Pattern

Each replacement follows the same pattern: the phrase `claude-opus-4-6` becomes `opus`. The surrounding context (e.g., "This overrides the agent's default model for this task") remains unchanged.

The metrics schema examples change from `"claude-opus-4-6"` to `"opus"` as an illustrative value.

## Rationale

Policy P-11 states model overrides are applied at spawn time. Currently the overrides use a specific Anthropic model ID rather than the tier alias. This bypasses Claude Code's `ANTHROPIC_DEFAULT_OPUS_MODEL` env var, which remaps what `opus` resolves to. Using the tier alias makes the env var mechanism work correctly.
