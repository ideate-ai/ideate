# WI-150: Fix watcher ignored pattern + integration test

## Context

The chokidar `ignored` pattern at `watcher.ts:24` is `/(^|[/\\])\../` which matches `.ideate/` itself (dot-prefixed directory), silencing all events. Three reviewers independently flagged this as the highest-priority defect in cycle 016 (code-quality C1, spec-adherence S2, gap-analysis G2).

## Implementation

### watcher.ts

Replace the ignored pattern at line 24. The only files that should be ignored inside `.ideate/` are the SQLite runtime files — otherwise a write to `index.db` would trigger a rebuild loop:

```ts
// Before:
ignored: /(^|[/\\])\../,

// After:
ignored: /index\.db(-wal|-shm)?$/,
```

This excludes `index.db`, `index.db-wal`, and `index.db-shm` from watch events. All other files (including `.DS_Store` etc.) are watched — chokidar's overhead for a few extra events is negligible.

### watcher.test.ts

Create `src/__tests__/watcher.test.ts` using vitest:

1. `beforeEach`: create a temp directory with a `.ideate/` subdirectory
2. Test: instantiate `ArtifactWatcher`, call `watch(tempIdeateDir)`, write a YAML file, use `vi.waitFor` or a promise-based listener to assert the `change` event fires within 2 seconds
3. Test: modify the YAML file, assert `change` event
4. Test: delete the YAML file, assert `change` event (unlink)
5. `afterEach`: call `watcher.close()`, remove temp dir

Use real filesystem operations, not mocks — the point is to test chokidar's actual behavior with the ignored pattern.
