### 4. Indexer Pipeline
- Startup: resolveArtifactDir → open DB → createSchema → rebuildIndex → start watcher
- Rebuild: walk YAML → hash check → two-pass insert (nodes + extension) → edge extraction → file ref extraction → stale deletion (CASCADE)
- FK OFF during insert phase, FK ON for cascade delete phase
- Cycle detection (Kahn's algorithm via Drizzle)

### 5. Tool Architecture
- ToolContext: { db, drizzleDb, ideateDir }
- 11 tools in 5 categories (context, query, execution, analysis, write)
- Hybrid query pattern: Drizzle for CRUD, raw SQL for recursive CTEs
- Write tools: YAML first (GP-8), then sync SQLite upsert

### 6. Graph Model
- 10 edge types with source/target type constraints
- Recursive CTE for depth > 1 traversal
- UNION deduplication for cycle protection
- Query modes: filter, traversal, combined

### 7. YAML Source of Truth
- GP-8 architecture
- MCP server as mandatory interface (P-6, P-26, P-32)
- Write path: skill → MCP write tool → YAML file → sync SQLite
- Read path: skill → MCP read tool → SQLite query → response

### 8. File Watcher
- Chokidar with 500ms debounce
- Triggers incremental rebuild on .ideate/ changes
- Hash-based skip for unchanged files
