# WI-191: Skill File Updates

## Tool-to-Skill Mapping

Each new tool needs MCP availability checks added to the skill files that consume it. The pattern matches the existing checks for `ideate_get_work_item_context` and `ideate_get_context_package`.

### Pattern
```markdown
**MCP availability check**: Look in your tool list for a tool whose name ends in `ideate_{tool_name}`.
If found:
1. Call it with ({parameters}).
2. Use the result. Skip the manual file reads below.
If not found, [manual fallback]:
[existing file-reading logic preserved as fallback]
```

## Skill-by-Skill Changes

### skills/execute/SKILL.md
- **Phase 2** (Completed Items Scan): Check for `ideate_get_execution_status`. If found, call it — returns completed/pending/blocked sets directly. Skip manual glob + file reads for resume detection.
- **Phase 10** (Journal Updates): Check for `ideate_append_journal`. If found, call it with skill="execute", date, entry_type, body. Skip manual file append.
- **Phase 11** (Status Reporting): Check for `ideate_get_project_status`. If found, use its response for milestone status reports. Skip manual aggregation.

### skills/review/SKILL.md
- **Phase 3.5** (Generate Review Manifest): Check for `ideate_get_review_manifest`. If found, use its response as the manifest content. Skip manual work item + incremental review file reading and table construction.
- **Phase 7.5** (Archive Completed Work Items): Check for `ideate_archive_cycle`. If found, call it with cycle_number. Skip manual file copy/move operations.
- **Phase 8** (Update Journal): Check for `ideate_append_journal`. Same pattern as execute.

### skills/refine/SKILL.md
- **Phase 3** (Load Prior Context): Check for `ideate_get_domain_state`. If found, use its response for domain policies + questions + cycle number. Skip manual reads of domains/*/policies.md and questions.md.
- **Phase 7h** (Work Items): Check for `ideate_write_work_items`. If found, call with new items array — server handles ID assignment, DAG validation, file writing. Skip manual YAML append + notes file creation.
- **Phase 7i** (Journal): Check for `ideate_append_journal`. Same pattern.

### skills/brrr/SKILL.md and skills/brrr/phases/*.md
- **brrr/phases/execute.md**: Same as execute skill — execution_status for resume, append_journal for journal.
- **brrr/phases/review.md**: Same as review skill — review_manifest, archive_cycle, append_journal.
- **brrr/phases/refine.md**: Check for `ideate_write_work_items` for targeted work item creation from finding groups. Check for `ideate_append_journal` for journal.
- **brrr/SKILL.md Phase 6c**: Check for `ideate_get_convergence_status`. If found, call with cycle_number — returns convergence verdict directly. Skip manual spec-adherence.md parsing.

## Verification
After all changes, grep all skill files for each of the 9 new tool names. Each must appear at least once as an MCP availability check. The 2 existing tools (get_work_item_context, get_context_package) are already present and should not be modified.
