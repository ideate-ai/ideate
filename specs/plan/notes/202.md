# WI-202: Plan Skill v3 Rewrite

## Scope

Full rewrite of `skills/plan/SKILL.md` for v3 architecture. This is the largest single skill change — the plan skill creates all initial artifacts and must now do so through MCP tools using YAML format into the .ideate/ directory structure.

## Key Changes

### Phase 1.1 — Directory Scaffolding
Current: Creates `{artifact-dir}/manifest.json`, `archive/incremental/`, `domains/`, `steering/`, `plan/` directories.
New: Calls an MCP tool to scaffold `.ideate/` directory structure with `config.json`. If no MCP tool exists for scaffolding, use `ideate_write_work_items` for work items and direct Write for `config.json` only (the one exception — bootstrapping the directory the MCP server needs).

### All Artifact Creation
Current: Uses Write tool to create markdown files (guiding-principles.md, constraints.md, interview.md, etc.).
New: Uses MCP write tools where available. For artifact types without dedicated write tools (principles, constraints, architecture), the plan skill writes YAML files to `.ideate/` directly — this is acceptable during the bootstrap phase since the MCP server may not be fully initialized yet.

**Important nuance**: The plan skill bootstraps the project. The MCP server reads from .ideate/, but .ideate/ doesn't exist until the plan skill creates it. During initial planning, direct YAML writes to .ideate/ are the bootstrap mechanism. After the first rebuild, MCP tools take over.

### Acceptance Criteria Rules (Phase 4.4 or similar)
Current: "Avoid criteria requiring human judgment... this should be rare."
New: Per GP-13, identify validation method for each criterion. Machine verification for objective criteria, human-in-the-loop for subjective. Both are first-class methods.

### MCP Error Pattern
Every ideate MCP tool call should be preceded by a check that the tool exists. If not:
```
Stop and report: "The ideate MCP artifact server is required but not available. Verify it is configured in .mcp.json and that `mcp/artifact-server/` has been built."
```

### Outpost References
Replace all "outpost" mentions with "external MCP servers." Preserve spawn_session as an external tool with availability check.

### Interview Format
Use the structured YAML interview format with `entries` array per the WI-199 changes.
