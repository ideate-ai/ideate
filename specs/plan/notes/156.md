# WI-156: Fix module-level mutable state in migration script

## Context

Code-reviewer S1 and WI-149 incremental review M1 both flagged six module-level `let` variables in `migrate-to-v3.ts` as unsafe for concurrent execution.

## Implementation

### scripts/migrate-to-v3.ts

1. Define a `MigrationContext` interface:
```ts
export interface MigrationContext {
  errors: string[];
  created: string[];
  ideateDir: string;
  sourceDir: string;
  dryRun: boolean;
  force: boolean;
}
```

2. Remove the six module-level `let` declarations
3. Create the context inside `runMigration`:
```ts
export function runMigration(opts: MigrationOptions): MigrationResult {
  const ctx: MigrationContext = {
    errors: [],
    created: [],
    ideateDir: opts.ideateDir,
    sourceDir: opts.sourceDir,
    dryRun: opts.dryRun ?? false,
    force: opts.force ?? false,
  };
  // ...
}
```

4. Thread `ctx` as the first parameter to every private helper function (`migrateGuidingPrinciples(ctx, ...)`, `migrateConstraints(ctx, ...)`, etc.)
5. Replace all bare references to `errors`, `created`, `ideateDir`, etc. with `ctx.errors`, `ctx.created`, `ctx.ideateDir`, etc.

### migrate.test.ts

Tests should pass without behavioral change. The `runMigration` function signature may change slightly (if it currently relies on module-level state for options) — update test calls accordingly. Export `MigrationContext` for test access if needed.
