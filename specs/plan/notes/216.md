# WI-216: Cost Reporting Script

## Design

`scripts/report-cost.sh` — focused on token and cost analysis.

### Sections:

1. **Per-Work-Item Token Cost** — Group entries by work_item field. Sum input_tokens + output_tokens per item. Sort descending by total. Table: Work Item | Total Tokens | Input | Output | Cache Read | Spawns.

2. **Per-Cycle Token Cost** — Group by cycle. Sum tokens per cycle with phase breakdown. Table: Cycle | Total Tokens | Plan | Execute | Review | Refine.

3. **Cost Trends** — Cycle-over-cycle total token comparison with trend indicator. If cycle_total_cost_estimate is available, include dollar amounts.

### Cost estimation
If entries have cycle_total_cost_estimate fields, use those directly. Otherwise, provide token counts only without dollar estimates. Do not hardcode token pricing — that changes frequently.
