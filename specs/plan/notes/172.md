# WI-172: Migration script fixes

## Overview

Addresses all cycle 018 significant and minor findings in the migration script. The script is `scripts/migrate-to-v3.ts` (TypeScript source) and `scripts/migrate-to-v3.js` (compiled JS that the test suite imports directly). Every change to `.ts` must be mirrored in `.js`.

**Q-41 resolution**: The migration script is a one-time conversion tool. Its purpose is to be called once to convert an existing v2 `specs/` artifact directory to the v3 `.ideate/` YAML format. It is not designed for incremental or repeated execution, though individual steps should be safe to re-run (idempotent per step).

### Q-62 — file_path: null in finding builders

**migrate-to-v3.ts**: In `migrateArchiveCycles` (and all finding builder functions it calls), set `file_path` to the relative path of the output YAML file instead of `null`.

The output path follows the pattern: `cycles/{NNN}/F-{WI}-{SEQ}.yaml` (relative to the `.ideate/` root). This should be the same path passed to `writeOutput`. Use the already-computed output path.

Example fix:
```typescript
// Before:
const finding = { ..., file_path: null, ... };

// After:
const outputPath = `cycles/${nnn}/F-${workItem}-${seq}.yaml`;
const finding = { ..., file_path: outputPath, ... };
writeOutput(outputPath, finding, ...);
```

Apply to both capstone findings (`migrateArchiveCycles`) and incremental findings. Update `.js` to match.

### cycleSeq ID collision fix

**migrate-to-v3.ts**: The `cycleSeq` counter resets to 0 at the start of each cycle directory, creating risk of ID collisions if two cycle directories contain findings with the same severity prefix and number.

Fix: make `cycleSeq` a global counter (initialized once before the cycle loop, not inside it), or use a compound ID scheme that includes the cycle number: `F-{NNN}-{WI}-{SEQ}`.

The simplest fix is making it a closure variable declared before the cycle loop:
```typescript
let globalFindingSeq = 0;
for (const cycleDir of cycleDirs) {
  // use globalFindingSeq++, not a per-cycle counter
}
```

Update `.js` to match.

### toYaml whitespace escape

**migrate-to-v3.ts**: In the `toYaml` function (or the YAML serialization utility), check if a string value starts with whitespace or a YAML-special character. If so, wrap it in double quotes before serialization.

```typescript
function quoteIfNeeded(value: string): string {
  if (/^[\s#&*!|>{%@`]/.test(value) || value.includes(': ') || value.includes(' #')) {
    return `"${value.replace(/"/g, '\\"')}"`;
  }
  return value;
}
```

Apply this check to all string values in the YAML output. Update `.js` to match.
