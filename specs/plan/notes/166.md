# WI-166: Spec + doc cleanup

## Context

Five minor findings from cycle 017 review require spec and documentation fixes. None involve source code logic — all are documentation inconsistencies or minor naming issues that accumulated across cycles 015-017.

## Changes required

### 1. notes/143.md — update journal layout spec

**File**: `specs/plan/notes/143.md`

The current content describes `journal.yaml` as a flat array file at `.ideate/journal.yaml`. This contradicts the per-entry file layout that `migrateJournal` actually implements (per the notes/157.md spec and user decision OQ-3 Option A). Update to describe the per-entry layout:

- Each journal entry is a separate YAML file at `.ideate/cycles/{NNN}/journal/J-{NNN}-{seq}.yaml`
- No single `journal.yaml` file at the root
- Remove any reference to `entries:` array or flat `journal.yaml`

### 2. notes/144.md — update JournalEntry schema

**File**: `specs/plan/notes/144.md`

The JournalEntry schema shows an `entries:` array in `journal.yaml`. Update to reflect individual files:

- Schema should describe a single JournalEntry YAML file, not a containing file with `entries:` array
- File path pattern: `cycles/{NNN}/journal/J-{NNN}-{seq}.yaml`
- Individual entry fields: `id`, `type: journal_entry`, `cycle`, `phase`, `date`, `title`, `content`

### 3. work-items.yaml — fix WI-144 edge type names

**File**: `specs/plan/work-items.yaml`

Find the WI-144 criteria section. The criterion text references edge type names `addresses` and `amends`. These were renamed to `addressed_by` and `amended_by` in WI-153. Update the criterion to use the current names.

Search for the criterion containing `idx_edges_composite` — that section in WI-144 should reference `addressed_by` and `amended_by`.

### 4. architecture.md — update directory structure

**File**: `specs/plan/architecture.md`

WI-155 changed `IDEATE_SUBDIRS` to use `cycles/` (flat) instead of `path.join("archive", "cycles")`. The architecture document still describes the old nested `archive/cycles` layout. Update the relevant section to show `cycles/` as the cycle-scoped artifact directory.

Read the architecture document first to find the relevant section (it will reference the `.ideate/` directory structure).

### 5. config.ts — rename CURRENT_SCHEMA_VERSION

**File**: `mcp/artifact-server/src/config.ts`

Rename the export `CURRENT_SCHEMA_VERSION` to `CONFIG_SCHEMA_VERSION`. This is the IdeateConfig JSON schema version constant (currently value 2). The name collision with `schema.ts`'s `CURRENT_SCHEMA_VERSION` (the SQLite user_version, currently value 3) causes confusion for consumers importing from both modules.

Update `config.test.ts` to reference `CONFIG_SCHEMA_VERSION`.

Do NOT rename anything in `schema.ts` — that constant serves a different purpose and its name is correct in context.

### 6. indexer.test.ts — fix archive/cycles fixture path

**File**: `mcp/artifact-server/src/__tests__/indexer.test.ts`

Around line 47, a test fixture creates a subdirectory called `archive/cycles` inside `.ideate/`. WI-155 established that the correct path is `cycles/` (flat, no `archive/` prefix). Update the fixture path from `archive/cycles` to `cycles`.

Read the file to find the exact line and surrounding context before editing.
