# WI-155: Update .ideate/ directory structure for cycle-scoped layout

## Context

D-60 and P-28 define the cycle-scoped directory structure. `createIdeateDir` in `config.ts` currently creates subdirectories that don't match the new layout.

## Implementation

### config.ts — createIdeateDir

Replace the current subdirectory list with:

```ts
const dirs = [
  "work-items",
  "principles",
  "constraints",
  "policies",
  "decisions",
  "questions",
  "modules",
  "research",
  "interviews",
  "cycles",
];
```

Remove old subdirectories that no longer apply (e.g., `archive/cycles` if present in the old list).

The `cycles/` directory is created empty. Cycle-specific subdirectories (`cycles/016/findings/`, `cycles/016/journal/`) are created by the skills that write to them (execute, review), not by `createIdeateDir`.

### config.test.ts

Update the `expectedDirs` array in the "creates .ideate/ directory with all expected subdirectories" test to match the new layout.
