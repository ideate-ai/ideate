# WI-171: MCP server performance and correctness

## Overview

This item bundles all cycle 018 significant and minor findings that touch `schema.ts`, `db.ts`, and `indexer.ts`. All changes are in the MCP artifact server core. File scopes are disjoint from WI-170 (watcher), WI-172 (migration script), and WI-173 (docs).

## Changes by Finding

### S2 â€” file_path indexes + pre-create prepared statements

**schema.ts**: Add `CREATE INDEX IF NOT EXISTS idx_{table}_file_path ON {table}(file_path)` for each typed table. These go at the end of `createSchema()`, after the existing table DDL. Tables requiring the index: work_items, findings, domain_policies, domain_decisions, domain_questions, guiding_principles, constraints, module_specs, research_findings, journal_entries, document_artifacts, interview_responses (even though being removed, the index creation will just be dropped along with the table).

Actually since interview_responses is being removed (Q-59 below), do not add an index for it.

**indexer.ts**: In `rebuildIndex`, the hash-check loop currently calls `db.prepare()` inside the per-file iteration. Move all `db.prepare()` calls for hash lookups to before the file loop. Store them in a map keyed by table name. Inside the loop, look up the pre-created statement.

### S3 â€” Drizzle/DDL constraint alignment

**db.ts**:
- `nodeFileRefs` table: add `primaryKey: { columns: [nodeFileRefs.nodeId, nodeFileRefs.filePath] }` to the Drizzle table definition
- `edges` table: add `unique: { columns: [edges.sourceId, edges.targetId, edges.edgeType] }` to the Drizzle table definition

These must align with the raw-SQL DDL in `schema.ts`. The existing `onConflictDoNothing()` and `onConflictDoUpdate()` calls in `indexer.ts` should continue to work correctly once Drizzle has the constraint metadata.

### S4 â€” detectCycles O(nÂ˛) â†’ index-pointer BFS

**indexer.ts**: In `detectCycles`, both BFS loops (Kahn's topological sort and the cycle-component BFS) use `queue.shift()` which is O(n). Replace with an index-pointer pattern:

```typescript
// Instead of: const current = queue.shift()!
// Use:
let head = 0;
// ...
const current = queue[head++];
```

This reduces complexity from O(nÂ˛) to O(n + e). Apply to both BFS loops in the function.
