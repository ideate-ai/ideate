# WI-192: Fix recursive CTE cycle protection and ambiguous column

## Changes

Two surgical fixes in `mcp/artifact-server/src/tools/query.ts`:

### Fix 1: UNION ALL → UNION (Q-75)
In the recursive CTE body (around line 427), change `UNION ALL` to `UNION` in both the outgoing and incoming recursive steps. This deduplicates visited nodes, preventing exponential row growth when traversing cyclic edges (e.g., mutual `references` between two nodes with `direction: "both"`).

### Fix 2: Alias n.id as node_id (Q-76)
In the CTE output SELECT (line ~430): change `n.id` to `n.id AS node_id`.
In the ORDER BY (line ~485): change `ORDER BY depth, id` to `ORDER BY depth, node_id`.
In the RawRow type: change `id: string` to `node_id: string`.
Update any code that reads `row.id` from traversal results to use `row.node_id`.

The filter mode (non-recursive path) is unchanged — it does not use the CTE.
