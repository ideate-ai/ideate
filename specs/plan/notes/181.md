# WI-181: Schema v7 DDL

## Key Changes from v6

The entire `createSchema` function is rewritten. The transaction creates:
1. `nodes` base table (8 common columns)
2. 12 extension tables (each with `id TEXT PRIMARY KEY REFERENCES nodes(id) ON DELETE CASCADE`)
3. `edges` table (FK to nodes for both source_id and target_id, no source_type/target_type)
4. `node_file_refs` table (FK to nodes for node_id, no node_type)
5. Indexes on nodes(type), nodes(file_path), edges(source_id, edge_type), edges(target_id, edge_type), node_file_refs(file_path), plus type-specific indexes on extension tables

## Extension Table Column Reference

Each extension table has `id TEXT PRIMARY KEY REFERENCES nodes(id) ON DELETE CASCADE` plus these type-specific columns:

- **work_items**: title TEXT NOT NULL, complexity TEXT, scope TEXT (JSON), depends TEXT (JSON), blocks TEXT (JSON), criteria TEXT (JSON), module TEXT, domain TEXT, notes TEXT
- **findings**: severity TEXT NOT NULL, work_item TEXT NOT NULL, file_refs TEXT (JSON), verdict TEXT NOT NULL, cycle INTEGER NOT NULL, reviewer TEXT NOT NULL, description TEXT, suggestion TEXT, addressed_by TEXT
- **domain_policies**: domain TEXT NOT NULL, derived_from TEXT (JSON), established TEXT, amended TEXT, amended_by TEXT, description TEXT
- **domain_decisions**: domain TEXT NOT NULL, cycle INTEGER, supersedes TEXT, description TEXT, rationale TEXT
- **domain_questions**: domain TEXT NOT NULL, impact TEXT, source TEXT, resolution TEXT, resolved_in INTEGER, description TEXT, addressed_by TEXT
- **guiding_principles**: name TEXT NOT NULL, description TEXT, amendment_history TEXT (JSON)
- **constraints**: category TEXT NOT NULL, description TEXT
- **module_specs**: name TEXT NOT NULL, scope TEXT, provides TEXT (JSON), requires TEXT (JSON), boundary_rules TEXT (JSON)
- **research_findings**: topic TEXT NOT NULL, date TEXT, content TEXT, sources TEXT (JSON)
- **journal_entries**: phase TEXT, date TEXT, title TEXT, work_item TEXT, content TEXT
- **metrics_events**: event_name TEXT NOT NULL, timestamp TEXT, payload TEXT (JSON)
- **document_artifacts**: title TEXT, cycle INTEGER, content TEXT

## Type-Specific Indexes

- `idx_work_items_domain ON work_items(domain)`
- `idx_findings_work_item ON findings(work_item, severity)`
- `idx_domain_policies_domain ON domain_policies(domain)`
- `idx_domain_questions_domain ON domain_questions(domain)`

## TypeScript Interface Changes

- `Edge` interface: remove `source_type` and `target_type` fields
- `NodeFileRef` interface: remove `node_type` field
- `ArtifactCommon` interface: unchanged (already defines the 8 common columns)
- All typed interfaces (WorkItem, Finding, etc.): unchanged
- `EDGE_TYPE_REGISTRY`: unchanged

## Schema Version

`CURRENT_SCHEMA_VERSION = 7`. The `checkSchemaVersion` function is preserved — it deletes the DB file on version mismatch and returns false, triggering a full rebuild.
