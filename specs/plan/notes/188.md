# WI-188: Analysis Tools

## ideate_get_convergence_status

### Parsing Rules (from brrr/SKILL.md Phase 6c)

**Step 1**: Read `{artifact_dir}/archive/cycles/{formatted_cycle}/spec-adherence.md`

**Step 2**: Search for machine-parseable verdict line:
- Pattern: `**Principle Violation Verdict**: Pass` or `**Principle Violation Verdict**: Fail`
- If found → use that verdict

**Step 3** (fallback): Find section by heading:
- Look for `## Principle Violation` or `## Guiding Principle` section
- If section body is `None.` or empty → Pass
- If section body contains `###` headings or `- ` list items (actual findings) → Fail

**Step 4** (fallback): If neither found → return `verdict: "unknown"` with `"unexpected format"` warning

**Finding counts**: Read `archive/cycles/{N}/summary.md`. Count bullet points under `## Critical Findings`, `## Significant Findings`, `## Minor Findings`, `## Suggestions`.

### Response Format
```yaml
cycle: 22
condition_a: pass  # zero critical+significant findings
condition_b: pass  # guiding principles satisfied
converged: true
findings:
  critical: 0
  significant: 0
  minor: 3
  suggestion: 1
principle_verdict: "pass"
principle_verdict_source: "spec-adherence.md line 47"
```

## ideate_get_domain_state

### Query Strategy
1. Read cycle number from `domains/index.md` (parse `current_cycle: N`)
2. For each domain (or filtered list): query `domain_policies` + `domain_questions` joined with `nodes`
3. Filter policies to active status, questions to open status
4. Optionally include decision count (not full decisions — too verbose)

### Response Format
```markdown
## Domain: workflow
Cycle: 22
Policies (5 active): P-1, P-18, P-21, P-22, P-23, P-24, P-30
Open questions (1): Q-72

## Domain: artifact-structure
Policies (4 active): ...
Open questions (2): Q-73, Q-74
```

## ideate_get_project_status

### Query Strategy
Aggregates from multiple sources:
1. Work item counts: `SELECT COUNT(*) FROM nodes WHERE type = 'work_item'`
2. Cycle number: from `domains/index.md` or document_artifacts
3. Completion counts: from execution_status logic (completed/pending/blocked)
4. Finding counts: from latest `archive/cycles/{N}/summary.md` (parse bullet counts)
5. Open questions: `SELECT COUNT(*) FROM domain_questions dq JOIN nodes n ON n.id = dq.id WHERE n.status = 'open' GROUP BY dq.domain`
6. Blocking items: from execution_status dependency analysis

### Response Format
```markdown
## Project Status

Plan: {N} work items total | Cycle {N} | {N} complete, {N} pending
Findings (current cycle): {N} critical, {N} significant, {N} minor
Blocking items: {list or "none"}
Open questions: {N} ({domain}×{count}, ...)
Last review: {date from latest cycle summary}
```
