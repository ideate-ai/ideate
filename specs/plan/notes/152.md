# WI-152: Schema additions — addressed_by, amended_by, user_version, files_failed

## Context

D-59 (reversed-direction scalar fields), D-60 (schema version pragma), code-quality M1/gap-analysis G3 (RebuildStats error reporting).

## Implementation

### schema.ts

1. Add `addressed_by TEXT` column to the `findings` table DDL (nullable, default NULL)
2. Add `amended_by TEXT` column to the `domain_policies` table DDL (nullable, default NULL)
3. Increment `CURRENT_SCHEMA_VERSION` from 2 to 3
4. In `createSchema`, after running all DDL, set: `db.pragma(\`user_version = ${CURRENT_SCHEMA_VERSION}\`)`
5. Export a `checkSchemaVersion(db, dbPath)` function that reads `db.pragma('user_version')` — if it doesn't match `CURRENT_SCHEMA_VERSION`, close the db, delete the file at `dbPath`, and return `false` (caller must reopen and recreate)

### indexer.ts

1. In `RebuildStats`, add `files_failed: number` and `parse_errors: string[]`
2. Initialize both to 0 and [] in `rebuildIndex`
3. When `safeParseYaml` returns null, increment `files_failed` and push `\`${filePath}: YAML parse error\`` to `parse_errors`
4. When `typeField` is falsy or `table` is undefined, increment `files_failed` and push a descriptive error
5. In `buildRow` for `findings` table case, add `addressed_by: toStrOrNull(doc.addressed_by)`
6. In `buildRow` for `domain_policies` table case, add `amended_by: toStrOrNull(doc.amended_by)`
7. Log parse errors at warn level: `console.warn(\`[indexer] ${stats.files_failed} files failed to parse\`)`

### index.ts

Add schema version check before `createSchema`:

```ts
const currentVersion = db.pragma('user_version', { simple: true }) as number;
if (currentVersion !== 0 && currentVersion !== CURRENT_SCHEMA_VERSION) {
  db.close();
  fs.unlinkSync(dbPath);
  // Reopen fresh
  db = new Database(dbPath);
  db.pragma("journal_mode = WAL");
  db.pragma("busy_timeout = 5000");
}
```

Note: `user_version = 0` means a fresh database (never had version set). Treat as compatible.
