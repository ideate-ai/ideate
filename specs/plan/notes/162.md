# WI-162: migrateArchiveCycles — extract work_item and verdict

## Context

`migrateArchiveCycles` in `scripts/migrate-to-v3.ts` (lines 951-1062) produces finding objects from review files (`code-quality.md`, `spec-adherence.md`, `gap-analysis.md`). The finding objects at lines 1034-1048 omit `work_item` and `verdict` — both are `NOT NULL` in the findings table DDL. The indexer's `buildRow` substitutes empty string `""` for missing values, so all migrated findings end up with `work_item = ""` and `verdict = ""` in the SQLite index.

This work item extracts both fields from the review file.

## What to extract and how

**`verdict`**: Every review file produced by the ideate review skill starts with a `## Verdict: {Pass|Fail}` line (the first non-empty line of the file). Parse it:

```typescript
function extractVerdict(content: string): string {
  const match = content.match(/^##\s+Verdict:\s*(Pass|Fail)/m);
  return match ? match[1] : '';
}
```

**`work_item`**: The review files in `archive/cycles/{NNN}/` are named after the work items they review (e.g., `code-quality.md`, `spec-adherence.md`, `gap-analysis.md`). These are capstone review files, not per-work-item reviews. For capstone files, `work_item` is the cycle number in a normalized form (e.g., `"cycle-017"`).

For `incremental/*.md` files (handled in WI-165), the filename prefix carries the work item number (e.g., `154-integrate-drizzle.md` → `WI-154`).

Capstone review files should use `work_item = "cycle-NNN"` derived from the parent cycle directory number.
