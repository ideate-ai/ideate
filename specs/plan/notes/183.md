# WI-183: Indexer Rewrite for Class Table Inheritance

## Key Architectural Changes

### Two-Pass Insert
Each YAML artifact inserts into TWO tables within the same transaction:
1. `nodes` (base table) — common fields: id, type, cycle_created, cycle_modified, content_hash, token_count, file_path, status
2. Extension table — type-specific fields only

Use `INSERT OR REPLACE` (via Drizzle `onConflictDoUpdate`) for both statements.

### Deferred FK During Rebuild
At the start of the rebuild transaction, before any inserts:
```typescript
db.pragma('defer_foreign_keys = ON');
```
This allows edges referencing target nodes that haven't been inserted yet. FK validation happens at COMMIT.

### Simplified Hash Check
Current: loops through 12 tables with 12 prepared statements.
New: single prepared statement on `nodes` table:
```sql
SELECT id, content_hash FROM nodes WHERE file_path = ?
```

### Simplified Stale Row Deletion
Current: `deleteStaleRows` iterates all typed tables, manually deletes edges and file_refs per node.
New:
```sql
DELETE FROM nodes WHERE id NOT IN ({keepIds})
```
ON DELETE CASCADE handles extension tables, edges, and node_file_refs automatically.

### buildRow Split
Current `buildRow` returns one row for the typed table.
New: split into two functions:
- `buildNodeRow(doc, content, filePath, hash)` → returns the 8-column nodes row
- `buildExtensionRow(table, doc)` → returns type-specific columns only

### Edge Extraction
`extractEdges` function signature changes: remove `_table` parameter (unused). `nodeType` is read from `doc.type` or from `nodes.type` via the already-inserted base row. The edge table no longer stores `source_type`/`target_type`, so `upsertEdge` signature drops those parameters.

### File Ref Extraction
`extractFileRefs` drops `nodeType` parameter. `upsertFileRef` drops `nodeType`.

### Cycle Detection
`detectCycles` is unchanged — it queries `edges` table for `depends_on` edges and runs Kahn's algorithm. The edge table structure is the same (just missing source_type/target_type which detectCycles never used).

## Removed Code
- `ALL_TYPED_TABLES` constant
- `hashCheckStmts` Map (12 prepared statements)
- `TYPE_TO_TABLE` mapping (replaced by import from db.ts)
- Manual cascade logic in `deleteStaleRows` (~20 lines)
