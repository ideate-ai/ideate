# WI-161: detectCycles traversal limit

## Context

`detectCycles` in `mcp/artifact-server/src/indexer.ts` (lines 423-503) implements Kahn's topological sort algorithm in TypeScript. It loads all `depends_on` edges from SQLite into memory with a single fixed SQL query, then processes them in JavaScript. It has no limit on the number of nodes or edges it will process.

On a large graph (e.g., thousands of work items with dense dependencies), this could load megabytes of data into memory and run for a significant time. Adding a pre-check guard prevents runaway processing.

Note: this is NOT a SQL CTE depth limit. The user's phrasing ("CTE traversal") was colloquial — the actual implementation is a TypeScript loop, not a recursive SQL query. The limit is an application-level count check before the algorithm runs.

## Target implementation

Add pre-checks before the algorithm runs:

```typescript
export const MAX_DEPENDENCY_NODES = 10_000;
export const MAX_DEPENDENCY_EDGES = 50_000;

export function detectCycles(db: Database): string[][] {
  const rows = db.prepare("SELECT source_id, target_id FROM edges WHERE edge_type = 'depends_on'").all() as { source_id: string; target_id: string }[];

  if (rows.length > MAX_DEPENDENCY_EDGES) {
    throw new Error(`detectCycles: edge count ${rows.length} exceeds limit ${MAX_DEPENDENCY_EDGES}`);
  }

  // Count unique nodes
  const nodeSet = new Set<string>();
  for (const row of rows) {
    nodeSet.add(row.source_id);
    nodeSet.add(row.target_id);
  }
  if (nodeSet.size > MAX_DEPENDENCY_NODES) {
    throw new Error(`detectCycles: node count ${nodeSet.size} exceeds limit ${MAX_DEPENDENCY_NODES}`);
  }

  // ... rest of Kahn's algorithm unchanged
}
```

The constants should be exported so tests and calling code can reference them.
