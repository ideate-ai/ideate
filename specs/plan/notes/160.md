# WI-160: deleteStaleRows Drizzle conversion

## Context

`deleteStaleRows` in `mcp/artifact-server/src/indexer.ts` was not converted during WI-154's Drizzle ORM integration. It iterates `ALL_TYPED_TABLES` (12 entries) using raw `db.prepare()` with template literal table names, violating two WI-154 acceptance criteria:
- "deleteStaleRows uses Drizzle delete().where() instead of raw SQL"
- "No column names or table names are interpolated as strings in SQL anywhere in indexer.ts"

This was caught by the cycle 017 capstone review. The incremental review for WI-154 missed it.

## Current implementation (indexer.ts:396-417)

```typescript
function deleteStaleRows(db: Database, table: string, seenIds: Set<string>): void {
  const rows = db.prepare(`SELECT id, file_path FROM ${table}`).all() as { id: string; file_path: string }[];
  for (const row of rows) {
    if (!seenIds.has(row.id)) {
      db.prepare(`DELETE FROM ${table} WHERE id = ?`).run(row.id);
    }
  }
}
```

`ALL_TYPED_TABLES` is the array of all table name strings. `deleteStaleRows` is called once per table type after indexing.

## Target implementation

`deleteStaleRows` needs to accept a Drizzle table reference instead of a string. The `TYPE_TO_DRIZZLE_TABLE` map (added in WI-154, in `db.ts`) maps table name strings to Drizzle table objects. The conversion pattern is:

```typescript
import { notInArray } from 'drizzle-orm';

function deleteStaleRows(db: DrizzleDb, tableRef: AnyDrizzleTable, keepIds: string[]): void {
  db.delete(tableRef).where(notInArray(tableRef.id, keepIds.length > 0 ? keepIds : [''])).run();
}
```

Notes:
- `notInArray` with an empty array is undefined behavior in Drizzle/SQLite — use `['']` as a sentinel when `keepIds` is empty (no rows should match `id = ''`)
- The call sites need updating to pass `TYPE_TO_DRIZZLE_TABLE[tableType]` instead of the string table name
- `seenIds: Set<string>` becomes `keepIds: string[]` (spread the Set or convert at call site)

## Scope boundary

`detectCycles` uses a fixed SQL query `SELECT source_id, target_id FROM edges WHERE edge_type = 'depends_on'` with no string interpolation. It is explicitly excluded from this work item per user decision (OQ-10). Do not modify `detectCycles`.
