# WI-151: Fix startup error handling in index.ts

## Context

`index.ts:19` calls `resolveArtifactDir({})` at module top level with no try/catch. Missing `.ideate/config.json` crashes with a raw stack trace. `rebuildIndex` at line 28 has the same problem. Code-reviewer S2, gap-analyst G7.

## Implementation

Wrap the startup sequence in try/catch blocks. On failure, log a human-readable message and exit cleanly:

```ts
let ideateDir: string;
try {
  ideateDir = resolveArtifactDir({});
} catch (err) {
  console.error(`[ideate-artifact-server] ${(err as Error).message}`);
  process.exit(1);
}

const dbPath = path.join(ideateDir, "index.db");
const db = new Database(dbPath);
db.pragma("journal_mode = WAL");
db.pragma("busy_timeout = 5000");
createSchema(db);

let stats: RebuildStats;
try {
  stats = rebuildIndex(db, ideateDir);
} catch (err) {
  console.error(`[ideate-artifact-server] Index rebuild failed: ${(err as Error).message}`);
  process.exit(1);
}

console.error(`[ideate-artifact-server] started, ${stats.files_scanned} files indexed`);
```

The startup log line goes to stderr (not stdout) because stdout is the MCP stdio transport.
