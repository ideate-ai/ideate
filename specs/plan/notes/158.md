# WI-158: Watcher + indexer test expansion — edge type extraction coverage

## Context

WI-148 incremental review M2 deferred edge extraction tests for all types except `depends_on`. Gap-analyst G6 noted `addresses` and `amends` (now `addressed_by` and `amended_by`) have no extraction tests. WI-150 fixes the watcher but the integration test (watcher → rebuild → database) is its own concern.

## Dependencies

- WI-150 (watcher fix must be in place for integration test)
- WI-152 (schema must have addressed_by/amended_by columns)

## Implementation

### indexer.test.ts — edge extraction tests

Add one test per auto-extracted edge type. Each test writes a minimal YAML fixture to a temp `.ideate/` directory, runs `rebuildIndex`, and asserts the correct edge exists in the `edges` table.

| Edge Type | Fixture | Expected Edge |
|---|---|---|
| `blocks` | work_item with `blocks: [WI-002]` | WI-001 → WI-002, type blocks |
| `belongs_to_module` | work_item with `module: auth` | WI-001 → auth, type belongs_to_module |
| `belongs_to_domain` | work_item with `domain: workflow` | WI-001 → workflow, type belongs_to_domain |
| `derived_from` | domain_policy with `derived_from: [GP-01]` | P-001 → GP-01, type derived_from |
| `relates_to` | finding with `work_item: WI-001` | F-001 → WI-001, type relates_to |
| `supersedes` | domain_decision with `supersedes: D-001` | D-002 → D-001, type supersedes |
| `addressed_by` | finding with `addressed_by: WI-150` | F-001 → WI-150, type addressed_by |
| `amended_by` | domain_policy with `amended_by: P-013` | P-012 → P-013, type amended_by |

### watcher.test.ts — integration test

Extend the WI-150 watcher test to verify the full chain:
1. Create temp `.ideate/` with a SQLite database
2. Start watcher
3. Write a YAML file to the watched directory
4. Wait for watcher event
5. Assert the database contains the new row (verifying `rebuildIndex` was triggered)

This is distinct from the WI-150 event-fires tests — it verifies the end-to-end path from file write to database update.
